% -------------------------------------------
% 1. Purpose:
%      Incubator files are working documents for developing and refining
%      paper content. Each file focuses on one aspect (architecture,
%      displays, structure, etc.) and can be compiled independently.
%
% 2. Section hierarchy:
%      \section{}        -> Top-level grouping (one per file)
%      \subsection{}     -> Major topic (each starts on a new page)
%      \subsubsection*{} -> Sub-topics within a subsection (unnumbered)
%
% 3. Page layout rules:
%      - Every \subsection EXCEPT the first gets a \clearpage before it
%
% 4. Discussion boxes:
%      - Wrapped in \fbox{\parbox{0.95\textwidth}{...}}
%      - Format: \textbf{> JL:} comment \\ \textbf{>> CC:} response
%      - Remove before final submission
%
% 5. Formatting conventions:
%      - \critical{} = red bold, \high{} = orange bold, \medium{} = blue bold
%      - [nosep] on itemize/enumerate
%      - Separator comments: % === (section), % --- (subsection), % . . . (sub)
%
% =============================================================================
